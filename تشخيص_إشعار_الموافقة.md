# 🔍 تشخيص: لماذا لا يصل إشعار الموافقة/الرفض للموظف؟

## ✅ تم إضافة رسائل تشخيص مفصلة

الآن عند الموافقة أو الرفض على طلب، ستظهر رسائل تفصيلية في Console توضح كل خطوة.

---

## 🧪 خطوات التشخيص

### الخطوة 1: افتح Console في المتصفح

1. اضغط **F12** في Chrome/Edge
2. اذهب لتبويب **Console**
3. امسح الرسائل القديمة (زر 🚫 أو Ctrl+L)

### الخطوة 2: وافق أو ارفض طلب إجازة

1. افتح `hr-management.html`
2. سجّل دخول كمدير
3. اذهب لقسم الطلبات
4. اضغط **✓ قبول** أو **✗ رفض** على أي طلب

### الخطوة 3: راقب رسائل Console

## ⚠️ المشاكل المحتملة وحلولها

### المشكلة 1️⃣: الدالة غير موجودة

**الرسالة:**
```
⚠️ sendLeaveApprovalNotification function not found
```

**السبب:** ملف `emailjs-notifications.js` لم يتم تحميله

**الحل:**
1. تأكد من وجود السطر في `hr-management.html`:
```html
<script src="emailjs-notifications.js"></script>
```
2. تأكد من وجود الملف في نفس المجلد
3. حدّث الصفحة بـ **Ctrl+Shift+R**

---

### المشكلة 2️⃣: EmailJS غير جاهز

**الرسالة:**
```
⚠️ EmailJS is not available - notification cannot be sent
```

**السبب:** مكتبة EmailJS لم تحمّل

**الحل:**
1. افتح `setup-emailjs-offline.html`
2. حمّل ملف `email.min.js`
3. ضعه في نفس مجلد `hr-management.html`
4. حدّث الصفحة

---

### المشكلة 3️⃣: لا يوجد إيميل للموظف

**الرسالة:**
```
⚠️ No employee email found - notification cannot be sent
Employee email: undefined
```

**السبب:** الموظف ليس له بريد إلكتروني

**الحل:**
1. افتح `admin.html`
2. اذهب لـ **📧 إدارة الإيميلات**
3. أضف بريد إلكتروني للموظف
4. أعد المحاولة

---

### المشكلة 4️⃣: خطأ في جلب بيانات الموظف

**الرسالة:**
```
⚠️ Could not fetch emails for approval notification
```

**السبب:** خطأ في قاعدة البيانات أو الموظف غير موجود

**الحل:**

تحقق من وجود الموظف:
```sql
SELECT employee_id, name, email, manager_id
FROM employees
WHERE employee_id = 'E001'; -- غيّر E001 برقم الموظف
```

إذا لم يظهر:
- تأكد من أن الموظف موجود في النظام
- تأكد من صحة `employee_id` في طلب الإجازة

---

### المشكلة 5️⃣: خطأ في إرسال البريد

**الرسالة:**
```
❌ Error sending approval notification: {...}
```

**الأسباب المحتملة:**

1. **Template غير موجود:**
```
Template not found
```
**الحل:** أنشئ قالب `template_leave_approval` في EmailJS Dashboard

2. **Service ID خطأ:**
```
Service not found
```
**الحل:** تحقق من `serviceID` في `emailjs-notifications.js`

3. **Public Key خطأ:**
```
Invalid public key
```
**الحل:** تحقق من `publicKey` في `emailjs-notifications.js`

## 🧪 اختبار سريع

### تحقق من البيانات الأساسية:

```sql
-- 1. تحقق من بيانات الموظف
SELECT 
    employee_id,
    name,
    email,
    manager_id
FROM employees
WHERE employee_id = 'E001'; -- غيّر بالرقم الوظيفي

-- يجب أن يظهر:
-- employee_id: E001
-- name: أحمد
-- email: ahmad@company.com ✅ (مهم جداً!)
-- manager_id: 1003
```

### تحقق من طلب الإجازة:

```sql
-- 2. تحقق من طلب الإجازة
SELECT 
    id,
    employee_id,
    employee_name,
    status,
    request_type,
    from_date,
    to_date
FROM leave_requests
ORDER BY created_at DESC
LIMIT 5;

-- يجب أن يظهر employee_id صحيح
```
