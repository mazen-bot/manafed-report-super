# 📋 دليل ميزة تأكيد استلام الإنذارات
## Warning Acknowledgement Feature Guide

---

## 📌 نظرة عامة

تم إضافة ميزة جديدة تتطلب من الموظف **تأكيد استلامه للإنذار** بشكل صريح. هذه الميزة تضمن:
- ✅ علم الموظف بالإنذار الصادر بحقه
- ✅ توثيق تاريخ ووقت استلام الإنذار
- ✅ متابعة الإدارة لحالة استلام الإنذارات

---

## 🎯 المزايا الرئيسية

### 1️⃣ للموظف
- يستطيع رؤية جميع إنذاراته
- يعرف الإنذارات التي تحتاج للتأكيد (مميزة باللون الأحمر)
- يستطيع تأكيد استلام الإنذار بضغطة زر واحدة
- يرى تاريخ التأكيد للإنذارات المؤكدة

### 2️⃣ للإدارة
- متابعة من استلم الإنذار ومن لم يستلم
- توثيق رسمي لتاريخ الاستلام
- دليل قانوني على علم الموظف بالإنذار

---

## 🖥️ كيفية الاستخدام

### للموظف: تأكيد استلام الإنذار

1. **تسجيل الدخول** إلى نظام الموارد البشرية
2. انقر على **"📋 إنذاراتي"** من القائمة الجانبية
3. ستشاهد جميع إنذاراتك مع حالة كل إنذار:
   - 🔴 **بانتظار التأكيد**: إنذار جديد يحتاج لتأكيد (خلفية حمراء)
   - 🟢 **تم التأكيد**: إنذار مؤكد سابقاً (خلفية خضراء شفافة)

4. **للإنذارات غير المؤكدة**:
   - انقر على زر **"✓ تأكيد استلام الإنذار"** الأخضر
   - ستظهر رسالة تأكيد
   - انقر **موافق** للتأكيد
   - سيتحول الإنذار لحالة "تم التأكيد" ✅

5. **بعد التأكيد**:
   - سيظهر تاريخ التأكيد أسفل الإنذار
   - لن تستطيع إلغاء التأكيد (إجراء نهائي)

## 📊 حالات الإنذار

### 🔴 بانتظار التأكيد (Pending)
```
┌────────────────────────────────────────┐
│ إنذار أول     ⚠ بانتظار التأكيد  2026/02/27 │
│ التأخر عن موعد العمل                  │
│ ────────────────────────────────────   │
│ [✓ تأكيد استلام الإنذار]             │
└────────────────────────────────────────┘
```
**الخصائص:**
- خلفية حمراء فاتحة (`#fff5f5`)
- شريط أحمر على الجانب
- علامة تحذير `⚠`
- زر تأكيد أخضر ظاهر

### 🟢 تم التأكيد (Acknowledged)
```
┌────────────────────────────────────────┐
│ إنذار أول     ✓ تم التأكيد      2026/02/27 │
│ التأخر عن موعد العمل                  │
│ ────────────────────────────────────   │
│ ✓ تم التأكيد بتاريخ 28/02/2026      │
└────────────────────────────────────────┘
```
**الخصائص:**
- خلفية خضراء فاتحة (`#f0fdf4`)
- شريط أخضر على الجانب
- علامة صح `✓`
- معلومات تاريخ التأكيد ظاهرة
- لا يوجد زر تأكيد (مؤكد مسبقاً)

## 🗄️ البيانات المخزنة

### حقول جدول `employee_warnings`

| الحقل | النوع | الوصف |
|------|------|-------|
| `id` | BIGSERIAL | معرف فريد للإنذار |
| `employee_id` | TEXT | رقم الموظف |
| `warning_level` | TEXT | مستوى الإنذار (تنبيه، إنذار أول، إنذار ثاني، إنذار نهائي) |
| `reason` | TEXT | سبب الإنذار |
| `issued_by` | TEXT | رقم المدير الذي أصدر الإنذار |
| `issued_by_name` | TEXT | اسم المدير |
| **`acknowledged`** | **BOOLEAN** | **✅ حالة التأكيد (true = مؤكد، false = غير مؤكد)** |
| **`acknowledged_at`** | **TIMESTAMP** | **📅 تاريخ ووقت التأكيد** |
| `created_at` | TIMESTAMP | تاريخ إنشاء الإنذار |
| `updated_at` | TIMESTAMP | تاريخ آخر تحديث |

## 🔒 الأمان والصلاحيات

### Row Level Security (RLS)

تم تطبيق سياسات أمان لضمان:

1. **القراءة**: الموظف يمكنه قراءة إنذاراته فقط
2. **الإضافة**: المدراء فقط يمكنهم إضافة إنذارات
3. **التحديث**: 
   - المدراء: يمكنهم تحديث جميع حقول الإنذار
   - الموظفون: يمكنهم تحديث `acknowledged` و `acknowledged_at` **لإنذاراتهم فقط**
4. **الحذف**: المدراء فقط

### التحقق في الكود

```javascript
// عند تأكيد الإنذار
.eq('id', warningId)
.eq('employee_id', currentUser.employee_id) // 🔒 أمان: التأكد من أن الموظف يؤكد إنذاره فقط
```

## 💻 الكود التقني

### دالة تأكيد الإنذار

```javascript
async function acknowledgeWarning(warningId) {
    if (!confirm('هل تؤكد استلامك لهذا الإنذار؟\nسيتم تسجيل تاريخ التأكيد.')) {
        return;
    }

    const supabase = window.supabaseInstance;

    try {
        const { error } = await supabase
            .from('employee_warnings')
            .update({
                acknowledged: true,
                acknowledged_at: new Date().toISOString(),
                updated_at: new Date().toISOString()
            })
            .eq('id', warningId)
            .eq('employee_id', currentUser.employee_id);

        if (error) throw error;

        showCustomSuccess('تم تأكيد استلام الإنذار بنجاح', '✓ تم التأكيد');
        await loadUserData(); // إعادة تحميل البيانات
    } catch (err) {
        console.error('خطأ في تأكيد الإنذار:', err);
        showCustomError('حدث خطأ في تأكيد الإنذار: ' + err.message);
    }
}
```

## ❓ الأسئلة الشائعة

### س: هل يمكن للموظف إلغاء التأكيد بعد إتمامه؟
**لا**. التأكيد إجراء نهائي ولا يمكن إلغاؤه. هذا لضمان قيمة قانونية للتأكيد.

### س: ماذا لو لم يؤكد الموظف استلام الإنذار؟
يبقى الإنذار بحالة "بانتظار التأكيد" مع إبراز بصري (لون أحمر) حتى يقوم الموظف بالتأكيد. يمكن للإدارة متابعة الموظفين الذين لم يؤكدوا.

### س: هل يمكن للمدير تأكيد الإنذار بالنيابة عن الموظف؟
تقنياً نعم (لأن المدراء لديهم صلاحية تحديث جميع الحقول)، لكن هذا غير موصى به لأسباب قانونية وإدارية.

### س: كم من الوقت يستغرق ظهور التأكيد؟
فوري. بمجرد الضغط على زر التأكيد، يتم التحديث مباشرة وإعادة تحميل الصفحة.

## 📝 ملاحظات مهمة

- ⚠️ **التأكيد لا يعني القبول**: تأكيد الاستلام لا يعني موافقة الموظف على الإنذار، بل فقط إثبات أنه علم به
- 📅 **التوثيق**: يتم حفظ تاريخ ووقت التأكيد تلقائياً في قاعدة البيانات
