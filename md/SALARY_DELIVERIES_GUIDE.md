## 🚀 خطوات التثبيت

### الخطوة 1️⃣: تطبيق قاعدة البيانات

1. افتح **Supabase Dashboard**
2. اذهب إلى **SQL Editor**
3. افتح ملف `sql/create_salary_deliveries.sql`
4. انسخ جميع محتويات الملف
5. الصقها في SQL Editor
6. اضغط **Run** أو **تنفيذ**

✅ **ما سيتم إنشاؤه:**
- جدول `salary_deliveries` - لتسجيل تسليم الرواتب
- جدول `monthly_salary_records` - لسجلات الرواتب الشهرية
- دوال مساعدة للتقارير والإحصائيات
- فهارس لتحسين الأداء
- سياسات أمان RLS

### الخطوة 2️⃣: التحقق من التثبيت

بعد تنفيذ السكريبت، تحقق من نجاح العملية:

```sql
-- تحقق من وجود الجداول
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('salary_deliveries', 'monthly_salary_records');
```

يجب أن تحصل على نتيجتين:
- `salary_deliveries`
- `monthly_salary_records`

## 📖 دليل الاستخدام

### 🔐 تسجيل الدخول

1. افتح **admin.html**
2. سجّل الدخول برقمك الوظيفي وكلمة المرور
3. تأكد من أن لديك صلاحية "إدارة الموظفين" أو "إدارة الكاش"

---

### 💵 تسجيل تسليم راتب موظف

#### من لوحة التحكم (admin.html):

1. **اختر "تسليم الرواتب"** من القائمة الجانبية
2. **ابحث عن الموظف:**
   - استخدم صندوق البحث لتصفية الموظفين
   - أو اختر من القائمة المنسدلة
3. **أدخل تفاصيل الراتب:**
   - 💵 **مبلغ الراتب**: أدخل المبلغ الذي سيتم تسليمه (مثال: 5000.00)
   - 📅 **الشهر**: اختر الشهر المراد تسليم راتبه
   - 📅 **تاريخ التسليم**: اختر تاريخ اليوم أو التاريخ الفعلي للتسليم
   - 🧾 **رقم سند القبض** (اختياري): أدخل رقم سند القبض (مثال: RCP-2026-001)
   - 📝 **ملاحظات** (اختياري): أي ملاحظات إضافية
4. **اضغط "💾 حفظ تسليم الراتب"**

✅ **النتيجة:**
- سيتم حفظ السجل في قاعدة البيانات
- سيظهر إشعار بنجاح العملية
- ستتحدث قائمة السجلات تلقائياً

---

### 📊 عرض سجلات التسليم

#### في لوحة التحكم (admin.html):

بعد تسجيل التسليم، ستجد قسم "📊 سجل التسليمات الأخيرة":

**الفلاتر المتاحة:**
1. **تصفية حسب الفرع**: عرض تسليمات فرع معين فقط
2. **تصفية حسب الشهر**: عرض تسليمات شهر معين فقط

**الأعمدة المعروضة:**
- الموظف
- الرقم الوظيفي
- الفرع
- المبلغ
- الشهر
- تاريخ التسليم
- من سلّمه (اسم المدير/الموظف الذي سجل التسليم)
- رقم السند
- الحالة (تم التسليم / معلق)
- إجراءات (حذف)

---

### 📈 تقرير الرواتب الشامل

#### فتح صفحة التقارير (salary-report.html):

يمكنك الوصول للتقرير عبر:
1. فتح الملف مباشرة: `salary-report.html`
2. أو إضافة رابط في قائمة التنقل

**مميزات الصفحة:**

#### 📊 الإحصائيات العامة (في الأعلى):
- 💰 **إجمالي الرواتب المُسلّمة**: مجموع كل الرواتب التي تم تسليمها
- 👥 **عدد الموظفين تم تسليمهم**: عدد الموظفين الفريد الذين استلموا رواتبهم
- ⏳ **عدد الموظفين المتبقي**: الموظفين الذين لم يستلموا بعد
- 📋 **إجمالي السجلات**: إجمالي عدد سجلات التسليم

#### 🔍 الفلاتر المتقدمة:
1. **الفرع**: تصفية حسب فرع معين
2. **الشهر**: تصفية حسب شهر معين
3. **بحث عن موظف**: ابحث باسم أو رقم الموظف
4. **الحالة**: تصفية حسب حالة التسليم (تم / معلق)

#### 📋 الجدول التفصيلي:
يعرض جميع سجلات التسليم مع:
- الترقيم التسلسلي
- معلومات الموظف
- تفاصيل المبلغ
- تاريخ التسليم
- من سلّمه
- رقم السند
- حالة التسليم

## 📊 التقارير والدوال المتقدمة

### الدوال المساعدة في قاعدة البيانات:

#### 1️⃣ إجمالي الرواتب المُسلّمة في شهر معين:

```sql
SELECT get_total_salaries_delivered('2026-03', 2026);
-- النتيجة: إجمالي المبلغ بالريال
```

#### 2️⃣ عدد الموظفين الذين استلموا في شهر معين:

```sql
SELECT get_employees_paid_count('2026-03', 2026);
-- النتيجة: عدد الموظفين
```

#### 3️⃣ عدد الموظفين الذين استلموا في فرع معين:

```sql
SELECT get_employees_paid_count('2026-03', 2026, 'الرياض');
-- النتيجة: عدد الموظفين في فرع الرياض
```

#### 4️⃣ قائمة الموظفين الذين لم يستلموا بعد:

```sql
SELECT * FROM get_unpaid_employees('2026-03', 2026);
-- النتيجة: جدول بمعلومات الموظفين الذين لم يستلموا
```

## 🔧 إعدادات إضافية (اختياري)

### إضافة حقل الراتب للموظفين

إذا أردت تسجيل الرواتب الشهرية مسبقاً:

```sql
-- إضافة عمود الراتب الأساسي لجدول الموظفين
ALTER TABLE employees 
ADD COLUMN IF NOT EXISTS base_salary NUMERIC(12,2) DEFAULT 0;

-- تحديث راتب موظف
UPDATE employees 
SET base_salary = 5000.00 
WHERE employee_id = 'E001';
```

---

### إنشاء سجلات الرواتب الشهرية تلقائياً

```sql
-- إنشاء سجل راتب شهري لموظف
INSERT INTO monthly_salary_records (
    employee_id, 
    employee_name, 
    branch_name, 
    base_salary, 
    allowances, 
    deductions, 
    net_salary, 
    salary_month, 
    salary_year, 
    payment_status
) VALUES (
    'E001',
    'محمد أحمد',
    'الرياض',
    5000.00,  -- الراتب الأساسي
    500.00,   -- البدلات
    200.00,   -- الخصومات
    5300.00,  -- الصافي = الأساسي + البدلات - الخصومات
    '2026-03',
    2026,
    'pending'
);
```

## ❗ حل المشاكل الشائعة

### المشكلة 1: "جدول salary_deliveries غير موجود"

**الحل:**
- تأكد من تنفيذ ملف `create_salary_deliveries.sql` في Supabase
- راجع خطوات التثبيت أعلاه

---

### المشكلة 2: لا تظهر البيانات في التقرير

**الحل:**
1. تأكد من وجود سجلات في قاعدة البيانات:
```sql
SELECT COUNT(*) FROM salary_deliveries;
```
2. اضغط زر "🔄 تحديث البيانات"
3. افحص Console في المتصفح (F12) للأخطاء

---

### المشكلة 3: لا يمكن تسجيل تسليم الراتب

**الحل:**
1. تأكد من تسجيل الدخول بحساب له صلاحية
2. تأكد من اختيار جميع الحقول المطلوبة:
   - اختيار موظف
   - إدخال المبلغ
   - اختيار الشهر
   - اختيار التاريخ
3. تحقق من اتصال الإنترنت

---

### المشكلة 4: خطأ في الصلاحيات (RLS Policy)

إذا ظهرت رسالة "Permission Denied":

```sql
-- التأكد من تفعيل السياسات
SELECT schemaname, tablename, policyname, permissive 
FROM pg_policies 
WHERE tablename IN ('salary_deliveries', 'monthly_salary_records');
```

إذا لم تظهر نتائج، نفّذ قسم السياسات من السكريبت مرة أخرى.
