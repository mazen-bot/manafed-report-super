# 🔧 حل مشكلة: "لا يمكنني إضافة الإيميل"

## 🎯 التشخيص السريع

### الخطوة 1: افتح أداة التشخيص
افتح ملف **test-email-diagnostic.html** في المتصفح

هذه الأداة ستفحص:
- ✅ اتصال Supabase
- ✅ وجود جدول employees
- ✅ وجود عمود email
- ✅ صلاحيات الكتابة

---

## 🚨 المشاكل الشائعة والحلول

### المشكلة 1️⃣: عمود email غير موجود

**الأعراض:**
```
❌ عمود البريد الإلكتروني غير موجود في قاعدة البيانات
```

**الحل:**

1. افتح **Supabase Dashboard** → **SQL Editor**
2. شغّل هذا الأمر:

```sql
ALTER TABLE employees
ADD COLUMN IF NOT EXISTS email TEXT;

CREATE INDEX IF NOT EXISTS idx_employees_email ON employees(email);
```

أو افتح ملف `add_email_column.sql` وشغّله كاملاً.

---

### المشكلة 2️⃣: خطأ في الصلاحيات (RLS)

**الأعراض:**
```
❌ permission denied for table employees
❌ new row violates row-level security policy
```

**الحل:**

شغّل هذا SQL في Supabase:

```sql
-- السماح بالقراءة
DROP POLICY IF EXISTS "Allow public read employees" ON employees;
CREATE POLICY "Allow public read employees" ON employees 
FOR SELECT USING (true);

-- السماح بالتحديث
DROP POLICY IF EXISTS "Allow public update employees" ON employees;
CREATE POLICY "Allow public update employees" ON employees 
FOR UPDATE USING (true);
```

---

### المشكلة 3️⃣: الزر لا يعمل في admin.html

**الأعراض:**
- الضغط على زر "إضافة" أو "تعديل" لا يفعل شيء
- لا تظهر نافذة prompt

**الحل:**

1. افتح Console في المتصفح (اضغط F12)
2. ابحث عن أخطاء JavaScript باللون الأحمر
3. إذا رأيت خطأ مثل:
   ```
   editEmployeeEmail is not defined
   ```
   حدّث الصفحة بـ **Ctrl + Shift + R** (تحديث قوي)

---

### المشكلة 4️⃣: البريد الإلكتروني "غير صحيح"

**الأعراض:**
```
❌ البريد الإلكتروني غير صحيح
```

**الحل:**

تأكد من صيغة البريد الإلكتروني:

✅ **صحيح:**
- ahmad@company.com
- mohammed.saleh@example.net
- emp123@domain.co.uk

❌ **خطأ:**
- ahmad@company (ناقص النطاق .com)
- @company.com (ناقص اسم المستخدم)
- ahmad company.com (ناقص @)
- ahmad @company.com (مسافة غير مسموحة)

## 📋 خطوات التشخيص اليدوي

### 1. تحقق من وجود العمود

في **Supabase SQL Editor**، شغّل:

```sql
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'employees' 
AND column_name = 'email';
```

**النتيجة المتوقعة:**
```
column_name | data_type
email       | text
```

إذا لم تظهر نتيجة = العمود غير موجود!

---

### 2. تحقق من الصلاحيات

في **Supabase SQL Editor**، شغّل:

```sql
SELECT * 
FROM pg_policies 
WHERE tablename = 'employees';
```

يجب أن ترى سياسات تسمح بـ:
- `SELECT` (القراءة)
- `UPDATE` (التحديث)

---

### 3. اختبار يدوي

في **Supabase SQL Editor**، جرّب:

```sql
-- اختبار القراءة
SELECT employee_id, name, email 
FROM employees 
LIMIT 3;

-- اختبار الكتابة (غيّر E001 برقم موظف حقيقي)
UPDATE employees 
SET email = 'test@example.com' 
WHERE employee_id = 'E001';
```
