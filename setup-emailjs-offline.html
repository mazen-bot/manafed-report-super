📧 إعداد EmailJS بدون الاعتماد على CDN

حل بديل عندما لا تتمكن من تحميل EmailJS من الإنترنت

ℹ️ اتبع هذه الخطوات إذا كان لديك رسالة: "EmailJS library did not load"

✅ الخطوة 1: تحميل ملف EmailJS

انقر على الزر أدناه لتحميل ملف EmailJS (يتطلب اتصال إنترنت مرة واحدة فقط):

📂 الخطوة 2: حفظ الملف في المجلد الصحيح

بعد تحميل الملف email.min.js:

  • افتح مجلد المشروع: d:\wep\manafed-report-super-main\
  • ضع الملف email.min.js في هذا المجلد
  • تحقق من وجود الملف بجانب hr-management.html

🔧 الخطوة 3: تحديث hr-management.html

افتح ملف hr-management.html بمحرر نصوص وابحث عن السطور التالية:

<script id="emailjs-script" src="https://unpkg.com/@emailjs/browser@4.3.0/dist/build/email.min.js" ... ></script>

استبدلها بـ:

<script src="email.min.js"></script>

👉 ملاحظة: احذف كل السطور التالية (fallbackEmailJSLoad) لأننا لن نحتاجهم مع الملف المحلي:

<script> // Fallback CDN loader if unpkg fails function fallbackEmailJSLoad() { ... } </script>

✨ الخطوة 4: اختبار التثبيت

بعد حفظ الملفات:

  • افتح hr-management.html في المتصفح
  • افتح Developer Tools (F12)
  • اذهب إلى Console tab
  • ابحث عن: ✅ EmailJS loaded from local file

💡 نصائح إضافية

  • في حالة الخطأ: تأكد من وضع الملف في المجلد الصحيح
  • مسح الـ Cache: اضغط Ctrl+Shift+Delete لمسح الـ Cache
  • إعادة التحميل: اضغط Ctrl+F5 (Hard Refresh)
  • فحص الملف: افتح email.min.js في محرر نصوص للتأكد من وجود كود
