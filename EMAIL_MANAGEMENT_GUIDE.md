# ✅ دليل إدارة البريد الإلكتروني من لوحة التحكم

## تم إضافة قسم جديد! 📧

تم إضافة قسم **"إدارة الإيميلات"** في صفحة لوحة التحكم (admin.html) لتسهيل إضافة وتعديل البريد الإلكتروني للموظفين.

---

## 📋 كيفية الاستخدام

### الخطوة 1: افتح صفحة لوحة التحكم

1. افتح ملف `admin.html` في المتصفح
2. سجّل دخول بحساب المدير
3. من القائمة الجانبية، اضغط على **📧 إدارة الإيميلات**

### الخطوة 2: إضافة/تعديل البريد الإلكتروني

ستظهر لك قائمة بجميع الموظفين مع حالة البريد الإلكتروني:

| الاسم | الرقم الوظيفي | الفرع | البريد الإلكتروني | الإجراءات |
|-------|--------------|-------|------------------|-----------|
| محمد أحمد | E001 | الرياض | ✓ ahmad@company.com | ✏️ تعديل |
| فاطمة علي | E002 | جدة | ✗ غير محدد | ➕ إضافة |

#### لإضافة بريد إلكتروني جديد:
1. اضغط على زر **➕ إضافة** بجانب اسم الموظف
2. أدخل البريد الإلكتروني في النافذة المنبثقة
3. اضغط **OK**
4. سيتم حفظ البريد تلقائياً ✅

#### لتعديل بريد إلكتروني موجود:
1. اضغط على زر **✏️ تعديل** بجانب اسم الموظف
2. عدّل البريد الإلكتروني في النافذة المنبثقة
3. اضغط **OK**
4. سيتم تحديث البريد تلقائياً ✅

### الخطوة 3: البحث عن موظف

استخدم مربع البحث أعلى القائمة للبحث بـ:
- اسم الموظف
- الرقم الوظيفي

```
🔍 البحث: محمد
```

ستظهر فقط النتائج المطابقة.

## ✅ التحقق من صحة البريد الإلكتروني

النظام يتحقق تلقائياً من صحة البريد الإلكتروني:

### صحيح ✅
```
ahmad@company.com
mohammed.saleh@example.net
employee123@domain.co.uk
```

### خطأ ❌
```
ahmad@company          (ناقص النطاق)
@company.com          (ناقص اسم المستخدم)
ahmad company.com     (ناقص @)
ahmad @company.com    (مسافة غير مسموحة)
```

إذا أدخلت بريد إلكتروني غير صحيح، ستظهر رسالة خطأ:
```
❌ البريد الإلكتروني غير صحيح. يرجى إدخال بريد إلكتروني صحيح مثل: user@example.com
```

## 🔧 المتطلبات الفنية

### 1. يجب إضافة عمود البريد الإلكتروني

إذا لم يكن عمود `email` موجوداً في جدول `employees`، قم بتشغيل هذا الأمر في Supabase SQL Editor:

```sql
-- إضافة عمود البريد الإلكتروني
ALTER TABLE employees
ADD COLUMN IF NOT EXISTS email TEXT;

-- إنشاء فهرس
CREATE INDEX IF NOT EXISTS idx_employees_email ON employees(email);
```

**أو استخدم الملف الجاهز:**
افتح ملف `add_email_column.sql` وشغّله في Supabase SQL Editor.

### 2. التحقق من الصلاحيات

تأكد من أن سياسات RLS في Supabase تسمح بتحديث جدول `employees`:

```sql
-- سياسة السماح بالتحديث
CREATE POLICY "Allow public update employees" ON employees 
FOR UPDATE USING (true);
```

## 🎨 واجهة المستخدم

### الجدول يعرض:
- ✅ علامة خضراء: الموظف لديه بريد إلكتروني
- ❌ علامة حمراء: الموظف ليس لديه بريد إلكتروني
- 🔍 مربع بحث: للبحث السريع
- ➕ زر إضافة: لإضافة بريد جديد
- ✏️ زر تعديل: لتعديل البريد الموجود

### رسائل النظام:
- ✅ **نجاح:** `✅ تم إضافة البريد الإلكتروني بنجاح`
- ✅ **تعديل:** `✅ تم تعديل البريد الإلكتروني بنجاح`
- ❌ **خطأ:** `❌ حدث خطأ في حفظ البريد الإلكتروني`

## 🔗 الربط مع نظام الإشعارات

بعد إضافة البريد الإلكتروني للموظفين، سيعمل نظام الإشعارات EmailJS بشكل كامل:

### الإشعارات التي سيتم إرسالها:
1. 📨 **طلب إجازة جديد** → يُرسل للموظف والمدير
2. ✅ **الموافقة على الإجازة** → يُرسل للموظف
3. ❌ **رفض الإجازة** → يُرسل للموظف
4. ⏰ **تذكير الشهادة الطبية** → يُرسل للموظف (قبل 3 أيام من الموعد)
5. ⚠️ **إنذار جديد** → يُرسل للموظف

**شرط مهم:** يجب أن يكون للموظف **و** المدير (manager_id) بريد إلكتروني حتى تُرسل الإشعارات.

## 📊 إحصائيات الاستخدام

### عرض عدد الموظفين حسب حالة البريد الإلكتروني:

يمكنك الاستعلام في Supabase SQL Editor:

```sql
-- عدد الموظفين الذين لديهم بريد إلكتروني
SELECT COUNT(*) as with_email
FROM employees 
WHERE email IS NOT NULL AND email != '';

-- عدد الموظفين الذين ليس لديهم بريد إلكتروني
SELECT COUNT(*) as without_email
FROM employees 
WHERE email IS NULL OR email = '';

-- قائمة الموظفين بدون بريد إلكتروني
SELECT employee_id, name, branch_name
FROM employees 
WHERE email IS NULL OR email = ''
ORDER BY name;
```

## 🛠️ استكشاف الأخطاء

### المشكلة: لا تظهر قائمة الموظفين

**الحل:**
1. تحقق من Console في المتصفح (F12)
2. تأكد من تحميل Supabase بنجاح
3. تحقق من وجود جدول `employees` في قاعدة البيانات

### المشكلة: خطأ عند حفظ البريد الإلكتروني

**الحل:**
1. تأكد من إضافة عمود `email` في جدول `employees`
2. تحقق من سياسات RLS في Supabase
3. تحقق من رسالة الخطأ في Console (F12)

### المشكلة: زر "إضافة" أو "تعديل" لا يعمل

**الحل:**
1. تحقق من Console للأخطاء (F12)
2. جرّب تحديث الصفحة (Ctrl+F5)
3. تأكد من تسجيل دخول صحيح

1. **البريد الإلكتروني مطلوب:** لإرسال الإشعارات، كل موظف يجب أن يكون له بريد إلكتروني

4. **البحث الفوري:** مربع البحث يعمل فوراً بدون حاجة للضغط على زر
